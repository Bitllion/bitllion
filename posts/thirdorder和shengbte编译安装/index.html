@bitllion

源码包下载网址: https://www.shengbte.org/downloads/

thirdorder

1.安装依赖

1.1编译spglib

参见官方手册 https://spglib.github.io/spglib/install.html

不要使用pypi上的！！！

下载 https://github.com/spglib/spglib/archive/master.zip

unzip master.zip && cd spglib-master && mkdir build && cd build
cmake -DCMAKE_INSTALL_PREFIX=/root/spglib ..
make -j $nproc
make install

cmake 需要版本大于3，天河二号可使用 module load cmake/3.8.1 加载cmake3的环境变量

声明环境变量：

export LD_LIBRARY_PATH=/root/spglib/lib64:$LD_LIBRARY_PATH

1.2编译scipy

下载 https://files.pythonhosted.org/packages/26/b5/9330f004b9a3b2b6a31f59f46f1617ce9ca15c0e7fe64288c20385a05c9d/scipy-1.8.1.tar.gz

安装scipy的依赖
pip install pythran-0.11.0-py3-none-any.whl 
pip install ply-3.11-py2.py3-none-any.whl 
pip install gast-0.5.3-py3-none-any.whl 
pip install beniget-0.4.1-py3-none-any.whl 
pip install pythran-0.11.0-py3-none-any.whl

whl 可在pypi官网搜到，或者创建一个相同python版本的虚拟环境（推荐conda），试用pip download

<package_name> 下载

编译scipy

python setup.py install --user

2.修改配置

vim 打开thirdorder解压下的文件setup.py

将10、13行 改成如下所示，其中spglib的lib库实际为lib64

INCLUDE_DIRS = ["/WORK/jnu_youdikuang_1/software/spglib/include"]
LIBRARY_DIRS = ["/WORK/jnu_youdikuang_1/software/spglib/lib64"]

还需要修改thirdorder_core.c 第559行（后续版本变动建议用vim搜索spglib.h）

将

#include "spglib/spglib.h"

修改为

#include "spglib.h"

编译安装到用户空间

python setup.py install --user

shengbte

1.编译

解压ShengBte

复制arch.makefile.example到Src里

cd Src && cp ../arch.example  arch.make

加载环境变量，需要重新设置下MKLROOT

module load intel-compilers@2018
unset MKLROOT
export MKLROOT=/WORK/app/intel/parallel_studio_xe_2018/compilers_and_libraries_2018.0.128/linux/mkl

编辑arch.make,如下：

export FFLAGS= -debug -O2 -masm=intel
export LDFLAGS=-L/WORK/jnu_youdikuang_1/software/spglib/lib64 -lsymspg
export MPIFC=mpif90
MKL=$(MKLROOT)/lib/intel64_lin/libmkl_lapack95_lp64.a -Wl,--start-group	\
$(MKLROOT)/lib/intel64_lin/libmkl_intel_lp64.a				\
$(MKLROOT)/lib/intel64_lin/libmkl_sequential.a				\
$(MKLROOT)/lib/intel64_lin/libmkl_core.a -Wl,--end-group -lpthread -lm
export LAPACK=$(MKL)
export LIBS=$(LAPACK)

编译报错，就更改下arch.make，比如原本的FFLAGS是FFLAGS=-traceback -debug -O2 -static_intel但2018不支持这种编译选项，按照编译器的提示更改为上面的

模板里的寻找的库文件 lib/em64t/ 实际lib/intel64_lin，按照具体环境修改

编译make

编译完后会在上级目录生成二进制ShengBTE

2.环境配置和测试

新建一个目录存放二进制

mkdir ~/software/ShengBTE
mv ShengBTE ~/software/ShengBTE

export PATH=~/software/ShengBTE:$PATH
